MODULE MANAGEMENT

Installing Modules

  • Modules can now be installed one of two ways:
  • Per project
  • Globally

Per-Project Installation

  1. Open Titanium Studio.
  2. Expand the project folder in Project Explorer or select the project in the App Explorer menu
  3. Drag the .zip file from the download location into the current project, placing it at the root of the project. Choose to "Copy Files"

Global Installation

  1. Locate your Titanium SDK installation location
  2. Drag the .zip file under the /Titanium folder. As an example, on OS X, the path might look like "/Library/Application Support/Titanium/module.zip"

Using Your Module

tiapp.xml Editor

  1. Double-click on the tiapp.xml file
  2. In the right-hand column, click the + to add a new module. It will show you a list of modules available in your global directory and in your project
  3. Once you've added a module, clicking the pencil (edit) icon allows you to switch versions if there are multiple version available
  4. If you wish to delete a module, click the x (delete) button to delete the module

EXTENSIBLE PUBLISHING PLATFORM

  • Ability for third-party providers to hook into our publishing workflow
  • App stores can register within the interface
  • Deployment from inside Titanium Studio to an online location
  • Knowledge of Eclipse plugin development is necessary to develop a plugin that extends the built-in Publish functionality
  • See Adding A Publish Provider

New Views

  • A Snippets view that provides easy access to the complete list of snippets shipped with Studio, as well as simple ways to filter, preview and edit the list
  • A Servers view for stopping, starting and restarting external servers used for preview

Fixes & Enhancements

  • Content assist on user-defined functions without file save
  • Content assist on both hover and in Dynamic Help
  • Type /** before a javascript function will add a documentation header
  • Create a function in one file, and use it in another
  • Access Open Declaration (also via the F3 key) to jump back to the source definition
  • Validate entire project (as opposed to just the current file) as part of the build step, with warn before launching if project has errors
  • No indexing or validating files under "builds" folder

More Notable Changes

  • XCode 4.3 Support
  • Studio 2.0 is now built on top of Eclipse 3.7.2
  • iOS and Android distribution more intuitive
  • Emulators and Simulators now support custom launch arguments
  • Right-click on a project to open the analytics for that application
  • The console now auto-locks if you scroll away from the bottom
  • Work on decreasing the amount of processes during startup and editing to improve the editor experience
